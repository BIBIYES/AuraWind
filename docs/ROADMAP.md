## 📝 决策记录

### 2025-11-16: 架构决策
- **决策**: 采用 MVVM 架构
- **原因**: 与 SwiftUI 完美配合，测试友好
- **影响**: 代码结构清晰，但学习曲线稍陡

### 2025-11-16: UI 框架选择
- **决策**: 使用纯 SwiftUI
- **原因**: 原生支持，性能最佳，未来兼容性好
- **影响**: 放弃部分 AppKit 特性，但整体收益大

## 🤝 协作流程

### 分支策略
- `main`: 稳定版本
- `develop`: 开发分支
- `feature/*`: 功能分支
- `hotfix/*`: 紧急修复

### 发布流程
1. 功能开发完成
2. 合并到 develop
3. 创建 release 分支
4. 测试和 Bug 修复
5. 合并到 main 并打标签
6. 发布到 TestFlight/App Store
